Question:
- If you create a constructor method to build a DVD by title then create a constructor method to build
a DVD by category. Does JAVA allow you to do this?

Answer: It’s not allowed in JAVA. Title and Category have the same data type (String), and Java differentiates constructors based on the number, type, and order of parameters—not by the parameter names. Because both constructors would accept exactly one String argument, they would share the exact same method signature. The Java compiler cannot determine which constructor to execute when you pass a string value.

------------------------------- LAB03 -------------------------------
Question 1: After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two
objects still remain?
Answer: Because object references are passed by value. The swap method only swaps local copies of the references, leaving the original objects and their titles unchanged.

Question 2: After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is
the title of the JungleDVD changed?
Answer: Because while the method receives a copy of the reference, that copied reference still points to the exact same object in memory as the original jungleDVD variable. When the changeTitle method uses that reference to modify the object's internal data (like calling a setTitle() method), it is mutating the actual object itself. Since the original jungleDVD variable points to that same object, it reflects the updated title.

Question 3: Is JAVA a Pass by Value or a Pass by Reference programming language?
Answer: JAVA is a pass by value programming language.
